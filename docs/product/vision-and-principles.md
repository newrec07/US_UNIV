# 제품 비전과 원칙

## 제품 정의

Admissions OS는 미국 대학 입시를 위한 가족용 Agentic AI 컨설팅 웹앱이다.
단순한 정보 대시보드가 아니라 학생에게 “지금 무엇을 해야 하는가”를 쉽고 명확하게
안내하는 행동 가이드다.

초기 사용자는 학생 2명과 부모 2명이며 모두 같은 정보를 볼 수 있다. 기준 페르소나는
NZ NCEA 과정에 재학 중인 미국 영주권자이며 STEM 생명과학과 pre-med에 관심이 있다.

## 원칙 1 — 행동 가이드이지 점수판이 아니다

> 달성률과 진행률은 감독자의 언어다. 학생에게는 스트레스가 될 수 있다.

- rating, risk, confidence, growthStage, completeness 같은 측정값은 다음 행동으로 번역한다.
- 첫 화면에는 학생이 지금 할 수 있는 1~3개의 행동만 보여준다.
- 행동은 가능한 한 30분 안에 시작하거나 완료할 수 있는 원자적 단위로 만든다.
- `critical_gap`, `empty`, `NOT_STARTED`는 빨간 점수가 아니라 “여기서 시작하자”로 표현한다.
- 학생은 모든 현황과 근거를 볼 수 있지만 원하지 않을 때 강제로 노출하지 않는다.

표현 계층:

- `action_layer`: 첫 화면의 행동
- `status_layer`: 사용자가 열어 보는 현황, rating, risk, 근거
- `system_layer`: 출처, 계산 버전, Agent 실행 정보

## 원칙 2 — 구조 내구성

- 늘어나는 종류는 열린 문자열 타입으로 확장하고 유한한 척도는 닫는다.
- 변하고 궤적이 의미 있으며 되돌아볼 값은 이력으로 저장한다.
- 현재 상태와 추세는 이력에서 계산한다.
- 측정은 각 도메인 블록이 담당하고 행동은 학생별 단일 Action 큐가 담당한다.
- 객관적 공유 사실은 정규화하고 관계는 이름이 아니라 ID로 참조한다.
- 출처와 신뢰도를 독립적으로 추적한다.
- 미래를 전부 미리 구현하지 않고 나중에 추가해도 구조가 깨지지 않는 자리를 만든다.

## 원칙 3 — 규칙과 LLM의 경계

- 일정, 금액, 우선순위, 완성도 같은 결정적 값은 규칙 엔진이 계산한다.
- LLM은 근거를 읽고 설명, 코칭, 요약, 구조화된 변경 제안을 만든다.
- LLM이 공식 데이터나 금액을 기억으로 채우지 않도록 한다.
- Agent 추론과 학생·부모·공식 문서가 확인한 사실을 구분한다.
- 근거가 없는 진단은 저장하지 않는다.

## 원칙 4 — 합격 예측을 하지 않는다

Admissions OS는 합격 또는 불합격을 예측하지 않는다.
개인별 합격 확률과 거짓 정밀도를 제공하지 않는다.

대신 다음을 제공한다.

- 대학이 공개한 지원·등록 현황과 출처
- 학생의 학업, 시험, 활동, 일정, 재정 상태
- 근거가 연결된 강점, 빈 곳, 준비 상태
- 학업·선호·재정 조건 비교
- 지금 실행할 다음 행동

## 원칙 5 — 민감 데이터 최소 노출

- 구조화된 학생·재정 데이터와 Messenger 의사결정은 서버에 저장한다.
- Claude API 키, 시스템 프롬프트, Agent 도구는 브라우저에 두지 않는다.
- 원본 세금 문서와 성적표 저장은 기능 구현 시 별도 보안 결정을 거친다.
- 외부 분석·로그 시스템으로 보내는 개인정보를 최소화한다.

## 원칙 6 — 인구통계는 맥락이지 쿼터가 아니다

- 학생의 인구통계 정보는 선택적 자기보고 맥락으로 취급한다.
- 대학의 공개 등록자 분포는 출처가 있는 참고 데이터로만 제공한다.
- 인종 쿼터나 인종 기반 합격 공식을 모델링하지 않는다.
- 학생의 배경은 에세이와 경험의 맥락으로 다룰 수 있지만 합격 판정값으로 바꾸지 않는다.
