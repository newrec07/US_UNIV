# ADR-001 — 가족 4명용 비공개 v1

- 상태: Accepted
- 결정일: 2026-06-22

## 맥락

초기 사용자는 학생 2명과 부모 2명이다. 다중 가족 SaaS를 처음부터 지원하면 조직,
테넌트 격리, 결제, 복잡한 권한 모델이 핵심 입시 기능보다 먼저 필요해진다.

## 결정

v1은 가족 4명용 비공개 웹앱으로 개발한다. 사용자 role은 표시할 수 있지만 초기에는
학생과 부모가 같은 데이터에 접근한다.

## 결과

- 핵심 입시 흐름을 먼저 검증할 수 있다.
- 도메인 계층은 미래의 다중 가족 확장을 막지 않도록 UI와 인증에서 분리한다.
- SaaS 전환 시 별도 ADR과 데이터 migration이 필요하다.
